Now, suppose x and y represent coordinates in a 2-by-2-square:

  • (-1, -1) is the upper-left corner
  • (0, 0) is the exact center
  • (1, 1) is the lower right corner

We can use the value of an expression as a color value for each point by considering -1 to represent ”no color” and +1 to represent ”maximum color”. In computer graphics terms, color values range from 0 to 255, so -1 maps on to 0 and 255 maps on to +1.

Most sufficiently complicated expressions have a good chance of giving ”interesting” visual output. We can generate deeply-nested expressions (8-10 levels of nesting seems to work well) completely randomly, see how they look, and keep the good ones.

The central task in this project is to randomly generate expressions using the four primitives. You will do this in the same way as the random sentences we created in class. using the concept of a transformation grammar. The starting symbol is E (standing for “expression”), and the available transformations are:

E -> E * E

E -> (E + E) / 2

E -> sin(pi * E)

E -> cos(pi * E)

E -> x

E -> y

If you do this recursively, as long as every E eventually transforms into either x or y, you will end up with a nested expression of the kind you need.  To decide whether to use the last two productions, the recursive function you create needs to have a depth argument, and it should only use those last two productions when depth=1, otherwise picking one of the other productions at random and recursively calling itself with (depth-1).

Program Specification 

When the program starts, it should create a large, square graphics window. I recommend you work initially with a 300x300 window, but the final program should use a window of size at least 800x800.

The main interface gives the user these options:

  • Set the minimum and maximum recursion depth.
  • Enter an expression themselves, causing it to be rendered as a greyscale image.
    The value of the expression for each x,y value will indicate the brightness of the corresponding pixel, with -1 as pure black (0 for all 3 colors) and +1 as pure white (255 for all 3 colors).
  • Generate a random greyscale image.
    In this case, the program needs to generate just one random expression and apply it to set all three (red, green, blue) color values.
  • Generate for a random color image.
    In this case, the program needs to generate three different random expressions, using one for each of the three color values.
  • Quit

There are two classes already built for your use:

  • The Expression class represents a mathematical expression, and can be initialized with a string of the correct form.
    The evaluate(double x, double y) method of an Expression object can be used to get the expression's value for a given pair of x,y values.
  • The Picture class represents a picture. It has a few methods you will need to call:
    • set(int pixelX, int pixelY, int r, int g, int b)
      This sets the pixel at a particular position to a specific color, specified by red(r), green(g) and blue(b) values between 0 and 255.
    • show()
      This updates the picture the user sees. (You would typically call this after doing set(...) for all pixels in the image.)
      If it hasn't been called previously, it also opens a new window to display the picture in.

The program as a whole lives in the RandomArtApp class, which already has a complete main(), along with a lot of other code "pre-written" for you. You should read through the entire file and note all of the tasks I have written in different places, marked with a "TO DO" comment.

Implementation

You should start your work on this project by using top-down design, thinking carefully about how to decompose the overall program. There are at least three aspects that are almost completely independent: The user interface; creating an expression; calculating and showing an image.

There are two specifically tricky aspects of the assignment to be aware of:

(1) Getting the recursive generation of expression to work in a reasonable way. The program needs to keep track of a minimum and maximum recursion depth and make sure any expression generated is within those bounds. For instance, if the minimum depth is 1 and the maximum depth is 2, it might generate expressions like these:

x
y
avg(x, x)
y∗x
sin(pi ∗ y)

                but NOT these:

avg(x, y ∗ x)
sin(pi ∗ cos(pi ∗ x))

(2) Re-scaling values back and forth between the range [-1, 1] and the appropriate ranges for pixel positions and color values. Make sure you test everything carefully!
